# Система рекомендаций по замене блюд (ML Project)

## Описание

Учебный проект по машинному обучению. Приложение анализирует блюда русской и европейской кухни и определяет, подходят ли они для конкретной диетической цели: похудения, набора массы, баланса/ЗОЖ или контроля сахара. Если блюдо не подходит, система рекомендует более подходящую альтернативу из той же категории.

## Технологии

- **Python**: Pandas, NumPy (обработка данных)
- **Machine Learning**: Scikit-learn (GradientBoostingClassifier, StandardScaler, GridSearchCV, NearestNeighbors)
- **Web Interface**: Streamlit (интерактивное веб-приложение)
- **Serialization**: Pickle (сохранение моделей, скейлера и выборок)

## Структура проекта

- `app.py` — код веб-приложения (запуск через Streamlit).
- `notebooks/01_eda.ipynb` — разведочный анализ данных.
- `notebooks/02_labeling.ipynb` — создание целевых переменных.
- `notebooks/03_preprocessing.ipynb` — кодирование, масштабирование, Train/Test split.
- `notebooks/04_modeling.ipynb` — обучение и сравнение моделей.
- `notebooks/05_knn.ipynb` — поиск похожего блюда на замену.
- `notebooks/06_pipeline.ipynb` — финальный пайплайн end-to-end.
- `data/russian_food_dataset.csv` — исходный датасет (400 блюд).
- `data/russian_food_labeled.csv` — данные с целевыми метками.
- `data/best_models.pkl` / `data/scaler.pkl` / `data/splits.pkl` / `data/feature_cols.pkl` — сохранённые модели, скейлер и выборки.

## Как запустить приложение

1. Установите необходимые библиотеки:

   ```
   pip install streamlit pandas scikit-learn
   ```

   ```
   streamlit run app.py или python -m streamlit run app.py

   ```

Приложение откроется в браузере (обычно [http://localhost:8501](http://localhost:8501)).

## Логика работы

1. **Датасет**: собран вручную на основе нутриционных таблиц calorizator.ru и dietadiary.com — 400 блюд русской и европейской кухни (17 категорий), поскольку существующие открытые датасеты (американские блюда, упакованные продукты, NLP-формат) не подошли для задачи.
2. **Целевые переменные**: созданы на основе процентилей (75-й/25-й, отдельно по типу приёма пищи):
   - *Похудение*: высокие калории ИЛИ высокий сахар ИЛИ высокие жиры.
   - *Набор массы*: низкие калории ИЛИ низкий белок ИЛИ низкие углеводы.
   - *Баланс/ЗОЖ*: высокий сахар ИЛИ высокий натрий.
   - *Контроль сахара*: высокий сахар ИЛИ высокие углеводы.
3. **Модель**: сравнены 4 классификатора (логистическая регрессия, дерево решений, случайный лес, градиентный бустинг); гиперпараметры подобраны через GridSearchCV с кросс-валидацией. Градиентный бустинг — лучшая модель для всех 4 целей (F1 от 0.947 до 0.986 на Test), обучается отдельно для каждой цели.
4. **Рекомендации**: при несоответствии цели алгоритм kNN ищет замену внутри той же категории — ближайшее по Евклидову расстоянию блюдо, которое уже соответствует выбранной цели.
